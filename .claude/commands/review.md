# /review - Code / Notebook Review

Revisar codigo ou notebook do KB (qualidade, padroes, seguranca).

## Quando Usar

Apos implementar ou editar codigo/notebook no KB.

## Checklist

### Qualidade
- [ ] Codigo/notebook legivel e organizado
- [ ] Nomes descritivos
- [ ] Explicacoes presentes em tutoriais
- [ ] Diagramas onde fizer sentido (notebooks)

### Padroes KB
- [ ] Notebook segue padrao (titulo, objetivo, diagrama, explicacoes, resumo)
- [ ] Sem emojis em codigo ou docs tecnicos
- [ ] Python com type hints quando aplicavel

### Seguranca
- [ ] Sem credenciais hardcoded
- [ ] Input validation quando aplicavel

## Output

Relatorio com issues (critico/medio/baixo) e recomendacoes. Nao fazer commit; apenas reportar.
